AgentAdam — what's new

A round of pilot fixes and new capabilities
v1.1.3
TL;DR — the agent now asks better questions at the start of every session, has a more accessible colour palette, adapts your wireframes to fit the template, picks the right layout per page, and validates work in Desktop as it goes. You don't have to install anything — updates flow through automatically.

1 Better session start

The agent now asks two things at the start of every session that were previously easy to miss:

  • The report title for the top-right text box on every page
  • The page title for each page as it's built (not just "Example page")

Same for visual subtitles — when the agent copies a visual from the template vocabulary, it will now either rewrite the subtitle to give real context, or remove it. No more placeholder text left behind.

2 New colour palette

Colours have been switched to an accessibility-driven blue-and-orange palette. This is easier for colour-blind viewers to read and works better against our corporate green branding.

Min and Max are now contextual — the agent will ask whether "high" is good, bad, or neutral for your metric. Higher sales? Blue. Higher wait time? Orange. Higher temperature? Grey.

Want the older green/red palette? Just ask — it's still available on request, though less accessible.

3 The template always wins on layout

Two things are sacred, and they don't compete:

Template wins on FORM
Zones, coordinates, colours, spacing, slicer widths, legend defaults. The mechanics of how the template looks.
You win on CONTENT
The visuals you want, the story you're telling, the data you want to show, the audience you're serving.

When you share a wireframe that doesn't quite fit the template, the agent will adapt the placement to fit our zones — and always explain what it changed and why before proceeding. No silent reshaping. No refusing to build. Just a heads-up so you can approve or push back.

4 Layout patterns — right shape per page

The agent now picks the right layout pattern for each page based on what it's for. Tell it the purpose of the page — the agent picks and explains why. You can override any choice.

5 SQL review reads your model directly

You no longer need to paste your SQL for a review. Just ask the agent to review the SQL for a specific table, and it'll read the query straight from your model.

6 Desktop validation as we go

Instead of building everything then checking at the end, the agent now pauses at natural checkpoints to ask you to open Power BI Desktop and confirm things render properly:

  • After the model phase — confirm the data loads
  • After the first page — confirm the layout renders correctly
  • Before final sign-off — one last check

Catches errors early instead of letting them stack up across 16 pages.

7 Cleaner layout maths

Page titles now get more breathing room. The slicer zone was narrowed to make the page title area 70% wider (1200px, up from 707px). Slicers still work the same way — up to 4 across the top bar, the rest go to the filter pane.

8 New Layouts tab in the extension

If you've installed the AgentAdam VS Code extension, there's a new ▦ Layouts tab showing all four layout patterns with mini wireframes and copy-paste prompts. Reference tab has also been refreshed with the new colours and slicer widths.

Extension update: Adam will share the v1.0.3 installer in Teams. Install it over the top of your current version — VS Code will replace the old one automatically.
